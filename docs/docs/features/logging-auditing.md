# 日志与审计

统一记录系统运行与用户操作，支持排错、审计与合规。

## 日志级别与位置

- 级别：`debug`、`info`、`warn`、`error`
- 位置：应用日志目录与任务/工作流历史记录
- 导出：支持按时间段或任务 ID 导出

## 审计内容

- 关键操作：设备控制、工作流执行、配置变更
- 变更记录：谁在何时进行了何种操作
- 可回溯：提供查询与过滤能力

## 排错流程

1. 明确故障现象与影响范围
2. 检查相关时间段的日志与错误栈
3. 结合任务/工作流历史定位节点问题
4. 复现与修复后记录原因与改动

## 最佳实践

- 合理设置 `LOG_LEVEL`，生产环境避免冗余调试日志
- 对导出的日志进行脱敏，遵守隐私与合规要求
