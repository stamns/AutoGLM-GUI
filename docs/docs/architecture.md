# 架构概览

本页介绍 AutoGLM-GUI 的整体架构、关键模块与数据流，帮助你理解系统如何协同工作与扩展。

## 组件与职责

- 桌面应用：提供图形界面、设备接入、工作流编排与实时控制
- 前端界面：基于 Web 技术实现的页面与交互逻辑
- 后端服务：处理任务编排、设备通信、模型/服务集成与日志审计
- 设备端：摄像/屏幕/传感数据采集与控制执行

## 数据流与交互

1. 设备数据上行：设备端将视频流、屏幕或传感数据发送至后端/桌面端
2. 任务与工作流：用户在界面创建任务或工作流，配置步骤与参数
3. 模型/服务调用：后端按需调用模型或外部服务生成结果或控制指令
4. 控制下行：指令下发至设备端执行，并回传状态与日志

## 关键依赖与边界

- 网络通信：本地网络/USB/WiFi，必要时支持穿透与权限控制
- 模型与服务：通过可配置的键值或环境变量进行集成
- 日志与审计：统一记录任务、工作流与控制操作，支持导出

## 可扩展性与插件

- 新设备类型：通过设备适配层扩展采集与控制能力
- 新模型/服务：在集成层增加驱动与配置项
- 工作流节点：按步骤抽象扩展新的操作节点与模板

## 安全与合规

- 权限与角色：对关键操作设置校验与审核
- 数据与隐私：明确采集与存储策略，支持脱敏与清理

## 典型场景

- 采集→分析→控制：通过工作流串联，实现自动化闭环
- 人机协同：在手动控制模式下即时干预并记录操作
